Angular

Google doesn't support Angular project integration to old AngularJS. They "force" you to upgrade to the new version ( more info).
There are some ways to do this but you will find some problems anyway:
  • No official support
  • Definetely not the easiest and cleanest approach (specially on big projects)
  • Problem to get new dev into a project like that
  • Development time. Probably faster refactoring the actual code and doing a normal upgrade to Angular

So what about if we can't upgrade?

Our approach here was create a totally separete new project. Both of them running at the same time but linked each other.
If you check app.components inside AngularJS project you will see that we are picking the style files from the Angular one (this could be done in the other way around also).
On the same way we linked the menu options (app.component.html) to Angular. This would be replaced using some environment variables to use the real links when we compile the project.
